Git-based builds
Connect a GitHub repo, pick a branch and Dockerfile. Moor builds and deploys your image.
Cron jobs
Schedule commands inside your containers. View run history, output, and trigger jobs manually.
Web terminal
SSH into your containers or the host machine directly from the browser. No port forwarding needed.
Environment variables
Manage env vars per project. Changes are injected on the next container start.
Live logs
Stream build output and container logs in real time. See exactly what your app is doing.
Server monitoring
CPU, memory, disk, and container stats at a glance. Know when your server needs attention.

Why moor?

Built for developers who outgrew Coolify's complexity but don't need Kubernetes.

Coolify

  • Heavy abstraction layer over Docker
  • Complex multi-service architecture
  • Opinionated build pipeline
  • Large resource footprint
  • Many features you'll never use

moor

  • Thin layer — you see the Docker underneath
  • Single binary, single container
  • Your Dockerfile, your rules
  • Minimal footprint (~50MB)
  • Only what you need, nothing more

Get started in seconds.

One file. One command. You're live.

# create a compose file and run
$ curl -fsSL moor.sh/install | sh
$ docker compose up -d
